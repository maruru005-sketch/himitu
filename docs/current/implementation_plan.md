# Phase 11: トレード支援機能の高度化（AI要約・日誌・連携）

## 目的
本フェーズでは、DayTrade Terminal v3 を単なるチャート・シグナル表示ツールから、意思決定をサポートし、トレードの振り返りを自動化する総合プラットフォームへと進化させます。

## 主な追加機能

### 1. AIニュース・決算要約パネル
- **概要**: 選択中の銘柄に関連する最新ニュースや適時開示情報を取得し、LLM（Gemini等）を用いて要約表示します。
- **詳細**:
  - バックエンドで Google News RSS や Yahoo Finance 等から情報を取得。
  - 「ポジティブ（買い材料）」「ネガティブ（売り材料）」「中立」を判定し、3行で要約。
  - フロントエンドの `info-panel` に新しいカードとして追加。

### 2. トレードノート（日誌）自動生成
- **概要**: エントリー（BUY）および決済（SELL/決済完了）の瞬間のチャート画像を自動保存し、履歴とともに振り返れるようにします。
- **詳細**:
  - フロントエンドで `canvas.toDataURL()` を使用し、その瞬間のチャートキャプチャを生成。
  - バックエンドに送信し、`files/logs/` ディレクトリに保存。
  - 週末などの振り返り用に、履歴一覧から当時の画像を表示できるモーダルを追加。

### 3. ブラウザ拡張機能連携（PoC）
- **概要**: 外部サイト（楽天証券、SBI証券、株探等）とのデータ連携を強化します。
- **詳細**:
  - 第一段階として、外部サイトから銘柄コードを抽出し、ワンクリックで本ターミナルの監視銘柄に追加するブックマークレットまたは簡易拡張機能のベースを作成。

---

## ユーザー承認が必要な事項

> [!IMPORTANT]
> AI要約機能を使用する場合、Gemini APIキーなどの外部APIキー設定が必要になります。これは `config.json` で管理するようにします。

> [!NOTE]
> チャート画像の保存には、フロントエンド側でのキャプチャ処理が必要です。大量のトレードを行う場合、ディスク容量を考慮して一定期間（例：1ヶ月）で古い画像を削除するロジックが必要になる可能性があります。

## 提案される変更点

### バックエンド (`files/server.py`) [MODIFY]
- `/api/news/<code>` エンドポイントの追加（ニュース取得とAI要約）。
- `/api/capture` エンドポイントの追加（画像の受信と保存）。
- Gemini API 連携用のユーティリティ追加。

### フロントエンド (`files/index.html`) [MODIFY]
- AIニュース要約カードの追加。
- `togglePosition` 実行時にキャプチャを送信する処理の追加。
- トレード履歴（日誌）を表示する新しいUIの構築。

### 設定ファイル (`files/config.json`) [MODIFY]
- `gemini_api_key` などの設定項目を追加。

## 検証プラン

1. **AI要約テスト**: 実際にニュースを取得し、LLMが適切なトーンで要約を出力するか確認。
2. **キャプチャテスト**: 画像がサーバー上の指定フォルダに正しく保存され、表示できるか確認。
3. **負荷テスト**: ニュース取得によるAPIのレイテンシがフロントエンドの操作感に影響しないか確認（非同期処理）。
