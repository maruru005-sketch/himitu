# Phase 10: 短期取引向けのおすすめ銘柄（スクリーニング）機能 実装計画

監視銘柄とは別に、短期トレードに向いている（ボラティリティ・出来高が高い）銘柄を自動で抽出し、画面上に表示する機能を追加します。

## > [!IMPORTANT] User Review Required
yfinance には日本株全体を一度にスキャンする（スクリーニング）機能が標準で用意されていません。全上場銘柄（約4000社）のデータをリアルタイムで並列取得しようとすると、APIの制限に引っかかったり、著しいパフォーマンス低下を招く恐れがあります。

**提案するアプローチ:**
1. **「スクリーニング用ユニバース（対象母集団）」の事前定義:**
   短期トレードで人気のある大型株・高ボラティリティ株（例: レーザーテック、ソフトバンクG、アドバンテスト、三菱UFJなど約30〜50銘柄）をバックエンド側で事前にリスト化しておきます。
2. **並列処理でのデータ取得と評価:**
   この数十銘柄に対して並列でデータを取得し、「直近の出来高急増」「値上がり率」「MACDのゴールデンクロス」などの条件に基づいてスコアリングし、上位5〜10銘柄を抽出します。

このアプローチで進めてよろしいでしょうか？あるいは、特定の外部サイトからのスクレイピング等をご希望でしょうか？（安定稼働の観点からは上記のアプローチを推奨します）。

## Proposed Changes

### バックエンド実装 (`server.py`)

#### [MODIFY] server.py
- スクリーニングの対象となる人気銘柄リスト（30〜50件程度）を定義する定数 `SCREENER_UNIVERSE` を追加。
- おすすめ銘柄抽出用 API エンドポイント `GET /api/recommended` を追加。
  - `SCREENER_UNIVERSE` の銘柄データを並列取得し、スコアリングロジック（出来高の増加率、日中値上がり率、テクニカル条件など）で上位数件をピックアップして返す。

### フロントエンド実装 (`index.html`)

#### [MODIFY] index.html
- 左側パネル (`#stock-list`) の上部に、表示を切り替えるための「タブ（監視銘柄 / おすすめ）」UIを追加。
- 「おすすめ」タブが選択された際、`/api/recommended` を呼び出し、結果をリスト表示する処理を追加。
- リスト内の銘柄をクリックした場合、監視銘柄と同様にチャート等が表示されるようにする。

## Verification Plan

### Automated/Manual Verification
- バックエンドを起動し、`/api/recommended` が正常なレスポンス（上位銘柄のリスト）を返すか確認する。
- フロントエンドでタブが正常に切り替わるか確認する。
- 「おすすめ銘柄」から銘柄を選択した際、チャートデータが正常に表示され、各種指標が計算されるかテストする。
