# Phase 9: 長期運用・利便性向上のための拡張 実装プラン

Phase 9で予定されている以下の4つのタスクを実装するための計画です。

## 1. データの永続化 (Persistence)
現在フロントエンド上で管理されているPOSITIONとSIGNAL LOGのデータをバックエンド側で保存・復元できるようにします。

- **バックエンド (`server.py`)**:
  - データをJSONファイル (`state.json` など) に保存するためのエンドポイント `/api/state` (GET/POST) を追加します。
- **フロントエンド (`index.html`)**:
  - アプリケーション起動時に `/api/state` からデータを取得し、POSITIONとSIGNAL LOGを復元します。
  - ポジションの更新（エントリー/決済）や、新しいシグナルが発生した際に、非同期でサーバーに状態を送信して保存します。

## 2. パフォーマンス最適化 (並列処理)
監視銘柄が増加した際のレスポンス低下を防ぐため、銘柄スキャンのデータ取得を並列化します。

- **バックエンド (`server.py`)**:
  - `get_stocks()` エンドポイント内で各銘柄の `fetch_data` を実行しているループ処理を、`concurrent.futures.ThreadPoolExecutor` を用いた並列処理に変更します。
  - これにより、20銘柄以上でも数秒以内でデータを取得できるようになります。

## 3. インジケーターの拡充 (MACDの描画)
現在バックエンドで計算されているMACDの値を、チャート上に表示します。

- **フロントエンド (`index.html`)**:
  - Canvasの描画領域を下部に拡張し、メインのローソク足チャートとは別の「サブチャート領域」を作成します。
  - そこに MACDライン、シグナルライン、ヒストグラムを描画する処理 (`drawChart` 関数内) を追加します。

## 4. トレード成績の統計表示
日々のトレード結果（決済済みポジションの損益）を集計し、UIに表示します。

- **フロントエンド (`index.html`)**:
  - 永続化されたデータ（決済履歴）をもとに、本日（または指定期間）の実現損益を計算します。
  - POSITIONパネルの近く、または設定モーダル等に「本日の損益合計: ＋〇〇円」といった表示エリアを追加します。
