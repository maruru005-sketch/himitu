# AI売買判断および比較機能の実装計画

既存のテクニカル指標ベースのシグナルに加え、Gemini AIによる売買判断を取得し、両者を比較できる機能を追加します。

## ユーザーレビューが必要な項目
- **AI判断の頻度**: リアルタイム（30秒ごと）にAIを呼び出すとAPIクォータを消費しすぎる可能性があるため、ボタンクリック時または特定の時間間隔での更新を想定しています。
- **比較の表示形式**: チャート上にAIの推奨アイコンを表示するか、サイドパネルに比較テキストを表示するか。

## 修正・追加予定の機能

### 1. バックエンド実装 (`files/server.py`)

#### [MODIFY] `files/server.py`
- Geminiへのプロンプト作成関数の追加:
  - 直近のチャートデータ（OHLC、RSI、BB、MACD、VWAP）をテキスト化して渡す。
  - 現在のニュース要約も考慮に含める。
  - 「買い・売り・ステイ」の判断と、その根拠を返却させる。
- 新しいAPIエンドポイント `/api/ai-advice/<code>` の作成:
  - 指定銘柄の最新状況をAIが分析して返す。

### 2. フロントエンド実装 (`files/index.html`)

#### [MODIFY] `files/index.html`
- **比較パネルの追加**:
  - 「ロジック判断」と「AI判断」を並べて表示するUI。
- **チャートへの反映**:
  - AIの判断結果（Buy/Sell）をロジックのシグナルとは別の色（例：紫色の星印など）でチャート上に描画。
- **AIアドバイス更新ボタン**:
  - ユーザーが任意のタイミングでAIの最新見解を取得できるボタン。

## 検証計画

### 自動テスト
- `server.py` の新しいAPIが期待通りのJSON（decision, reason）を返すか確認。

### 手動確認
- 実際に銘柄を選択し、ロジックのシグナルとAIの判断がどのように分かれるか、UI上での比較が直感的かを確認。
- ニュースがある場合とない場合でAIの判断が変わるかテスト。
